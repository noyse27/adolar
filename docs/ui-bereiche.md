# Verbindliche Namen der Adolar-Web-Oberfläche

Diese Begriffe gelten für Besprechungen, Fehlermeldungen und neue Layout-Wünsche.

| Name | Bedeutung |
|---|---|
| **Kopfzeile** | Oberste Leiste mit Adolar-Logo, Statistik, Downloadkorb und Benutzermenü. |
| **Seitenleiste** | Gesamter linke Bereich mit Filtern, Playlists und Wartung. |
| **Filterbereich** | Oberer Teil der Seitenleiste mit Interpret-, Titel-, Album-, Genre-, Zeit- und Formatfiltern. |
| **Playlistbereich** | Abschnitt „Playlists“ in der Seitenleiste: Editor-Button, System-Playlists und persönliche Playlists. |
| **Wartungsbereich** | Unterer Teil der Seitenleiste mit Monitor, Scan, BPM, Last.fm und Datensicherung. |
| **Such- und Aktionsleiste** | Oberste Leiste des Inhaltsbereichs mit Volltextsuche, Sortierung und Radio. |
| **Listen-Kopf** | Schmale Leiste unter der Suche mit Trefferzahl, Playlist-Export und „Player öffnen“. |
| **Hauptbereich** | Mittlerer Inhaltsbereich, in dem die aktuelle Trackliste angezeigt wird. |
| **Trackliste** | Die einzelnen Trackzeilen innerhalb des Hauptbereichs. |
| **Seitennavigation** | Leiste unter der Trackliste zum Blättern zwischen Ergebnisseiten. |
| **Playerleiste** | Dauerhaft sichtbare untere Leiste mit aktuellem Track und Wiedergabesteuerung. |
| **Playeransicht** | Große Ansicht, die über „Player öffnen“ aufgerufen wird. |
| **Dialog** | Überlagerndes Fenster für eine begrenzte Aufgabe, zum Beispiel Playlist Editor, Monitor oder Einstellungen. |

## Playlist Editor

| Name | Bedeutung |
|---|---|
| **Editor-Kopf** | Titel und Schließen-Aktion oben. |
| **Editor-Werkzeuge** | Linke obere Spalte mit Track-Suche, Regeln und Import. |
| **Editor-Trefferliste** | Linke untere Liste der gefundenen Tracks. |
| **Editor-Playlist** | Rechte Trackliste mit der aktuellen Zusammenstellung und Reihenfolge. |
| **Auffüllbereich** | Bereich unter der Editor-Playlist für Zufallstracks. |
| **Editor-Aktionsleiste** | Unterste Leiste mit Export, Abbrechen und Speichern. |
| **Speicherdialog** | Zweiter Dialog für Playlistname und Typ „Smart“ oder „Statisch“. |

Beispiel: „Bitte den Export-Button im **Listen-Kopf** rechts neben **Player öffnen** breiter machen.“
